タスク

Locoのタスクは、アプリケーションの特定の側面を処理するためのアドホックな機能として機能します。データの修正、メール送信、ユーザーの削除、顧客の注文更新など、各シナリオに専用のタスクを作成することで、柔軟で効率的なソリューションを提供します。タスクは手動で実行することも、タスクをスケジュールして実行することもできます。

タスクを作成することには、いくつかの理由で価値があります。

  • 手作業の自動化: タスクは手動プロセスを自動化し、繰り返し行う作業を効率化します。
  • 慣れ親しんだコンポーネントの活用: アプリケーションのモデル、ライブラリ、および既存のロジックをタスク内で活用できます。
  • UI開発の排除: タスクはユーザーインターフェースの構築を必要とせず、バックエンド操作のみに焦点を当てます。
  • UI自動化の可能性: 必要に応じて、Jenkinsのようなジョブ実行ツールと統合することで、タスクをUIで自動化することも可能です。

各タスクは、CLIのyargs解析された出力を利用して、コマンドライン引数をフラグに解析するように設計されています。

CLIジェネレータでタスクを作成する

Locoは、プロジェクトに接続されたスタータータスクの作成を簡素化するための便利なコードジェネレータを提供します。タスクを生成するには、次のコマンドを使用します。

タスクを生成する:

Generate a Task based on the given name

Usage: demo_app-cli generate task [OPTIONS] <NAME>

Arguments:
  <NAME>  Name of the thing to generate

Options:
  -e, --environment <ENVIRONMENT>  Specify the environment [default: development]
  -h, --help                       Print help
  -V, --version                    Print version

タスクを実行する

前のステップで作成したタスクを、次のコマンドを使用して実行します。

cargo loco task <TASK_NAME>

パラメータ付きでタスクを実行する

タスクにパラメータを渡すには、コマンドに「key:value」のリストを追加します。

[PARAMS]...  Task params (e.g. <`my_task`> foo:bar baz:qux)`

cargo loco task <TASK_NAME> [PARAMS]...

次に、タスクの run メソッドで cli_arg を使用してその値を使います。

async fn run(&self, app_context: &AppContext, vars: &task::Vars) -> Result<()> {
    let foo = vars.cli_arg("foo");
    Ok(())
}

すべてのタスクを一覧表示する

実行されたすべてのタスクのリストを表示するには、次のコマンドを使用します。

cargo loco task

手動でタスクを作成する

Locoでタスクを手動で作成することを好む場合は、次の手順に従うことができます。

1. タスクファイルを作成する

まず、src/tasks パスの下に新しいファイルを作成します。たとえば、example.rs という名前のファイルを作成しましょう。

use loco_rs::prelude::*;

pub struct Foo;
#[async_trait]
impl Task for Foo {
    fn task(&self) -> TaskInfo {
        TaskInfo {
            name: "foo".to_string(),
            detail: "run foo task".to_string(),
        }
    }
    async fn run(&self, _app_context: &AppContext, _vars: &task::Vars) -> Result<()> {
        Ok(())
    }
}

2. mod.rs でファイルを読み込む

次に、src/tasks フォルダ内の mod.rs ファイルで、新しく作成したタスクファイルが読み込まれていることを確認します。

3. Appフックでタスクを登録する

Appフックの実装（例: App 構造体）で、register_tasks 関数にタスクを登録します。

// src/app.rs

pub struct App;
#[async_trait]
impl Hooks for App {
    ...

    fn register_tasks(tasks: &mut Tasks) {
        tasks.register(tasks::example::ExampleTask);
    }

    ...
}

これらの手順により、ExampleTask のような手動で作成したタスクが Loco のタスク管理システムに統合されます。
