認証

ユーザーパスワード認証

Locoはユーザー認証プロセスを簡素化し、新しいウェブサイトを迅速にセットアップできるようにします。この機能は時間を節約するだけでなく、アプリケーションのコアロジックの作成に集中できる柔軟性も提供します。

認証設定

auth機能はライブラリのデフォルトとして提供されます。必要に応じてオフにし、認証を手動で処理することもできます。

SaaSアプリを始める

loco cliを使用してアプリを作成し、SaaS app (with DB and user auth)オプションを選択してください。

組み込みの認証コントローラーを探索するには、次のコマンドを実行します。

$ cargo loco routes
 .
 .
 .
[POST] /api/auth/forgot
[POST] /api/auth/login
[POST] /api/auth/register
[POST] /api/auth/reset
[GET] /api/auth/verify
[GET] /api/auth/current
 .
 .
 .

新規ユーザー登録

/api/auth/registerエンドポイントは、アカウント確認のためのemail_verification_tokenを持つ新規ユーザーをデータベースに作成します。ユーザーには確認リンク付きのウェルカムメールが送信されます。

Curlリクエストの例:
curl --location '127.0.0.1:5150/api/auth/register' \
     --header 'Content-Type: application/json' \
     --data-raw '{
         "name": "Loco user",
         "email": "user@loco.rs",
         "password": "12341234"
     }'

セキュリティ上の理由から、ユーザーが既に登録されている場合、新規ユーザーは作成されず、ユーザーのメールアドレス詳細を公開することなく200ステータスが返されます。

ログイン

新規ユーザー登録後、以下のリクエストを使用してログインしてください。

Curlリクエストの例:
curl --location '127.0.0.1:5150/api/auth/login' \
     --header 'Content-Type: application/json' \
     --data-raw '{
         "email": "user@loco.rs",
         "password": "12341234"
     }'

レスポンスには、認証用のJWTトークン、ユーザーID、名前、および検証ステータスが含まれます。

{
    "token": "...",
    "pid": "2b20f998-b11e-4aeb-96d7-beca7671abda",
    "name": "Loco user",
    "is_verified": false
}
  • トークン: 認証エンドポイントへのリクエストを可能にするJWTトークン。設定ドキュメントを参照して、デフォルトのトークン有効期限をカスタマイズし、環境間でシークレットが異なることを確認してください。
  • pid - 新規ユーザー作成時に生成される一意の識別子。
  • 名前 - アカウントに関連付けられたユーザー名。
  • 検証済み - ユーザーがアカウントを検証したかどうかを示すフラグ。

アカウント確認

ユーザー登録時、確認リンク付きのメールが送信されます。このリンクにアクセスすると、データベースのemail_verified_atフィールドが更新され、ログインレスポンスのis_verifiedフラグがtrueに変わります。

Curlリクエストの例:

curl --location --request GET '127.0.0.1:5150/api/auth/verify/TOKEN' \
     --header 'Content-Type: application/json'

パスワードリセットフロー

パスワードを忘れた場合

forgotエンドポイントはペイロードにユーザーのメールアドレスのみを必要とします。パスワードリセットリンク付きのメールが送信され、データベースにreset_tokenが設定されます。

Curlリクエストの例:
curl --location '127.0.0.1:5150/api/auth/forgot' \
     --header 'Content-Type: application/json' \
     --data-raw '{
         "email": "user@loco.rs"
     }'

パスワードのリセット

パスワードをリセットするには、forgotエンドポイントで生成されたトークンと新しいパスワードを送信します。

Curlリクエストの例:
curl --location '127.0.0.1:5150/api/auth/reset' \
     --header 'Content-Type: application/json' \
     --data '{
         "token": "TOKEN",
         "password": "new-password"
     }'

現在のユーザーを取得

このエンドポイントは認証ミドルウェアによって保護されています。

curl --location --request GET '127.0.0.1:5150/api/auth/current' \
     --header 'Content-Type: application/json' \
     --header 'Authorization: Bearer TOKEN'

認証済みエンドポイントの作成

認証済みエンドポイントを確立するには、loco_rsライブラリからcontroller::middlewareをインポートし、認証ミドルウェアを関数エンドポイントのパラメータに組み込みます。

Rustの以下の例を考慮してください。

use axum::{extract::State, Json};
use loco_rs::{
    app::AppContext,
    controller::middleware,
    Result,
};

async fn current(
    auth: middleware::auth::Auth,
    State(ctx): State<AppContext>,
) -> Result<Response> {
    let user = users::Model::find_by_pid(&ctx.db, &auth.claims.pid).await?;
    /// Some response
}

API認証

新しいアプリを作成する

今回は、loco cliを使用してRESTアプリを作成し、Rest appオプションを選択してください。 新しいアプリを作成するには、以下のコマンドを実行し、指示に従ってください。

$ loco new

組み込みの認証コントローラーを探索するには、次のコマンドを実行します。

$ cargo loco routes
 .
 .
 .
[POST] /api/auth/forgot
[POST] /api/auth/login
[POST] /api/auth/register
[POST] /api/auth/reset
[GET] /api/auth/verify
[GET] /api/auth/current
 .
 .
 .

新規ユーザー登録

/api/auth/registerエンドポイントは、リクエスト認証のためのapi_keyを持つ新規ユーザーをデータベースに作成します。api_keyは今後のリクエストでの認証に使用されます。

Curlリクエストの例:

curl --location '127.0.0.1:5150/api/auth/register' \
     --header 'Content-Type: application/json' \
     --data-raw '{
         "name": "Loco user",
         "email": "user@loco.rs",
         "password": "12341234"
     }'

新規ユーザー登録後、新しいユーザーのapi_keyがデータベースに表示されていることを確認してください。

API認証を用いた認証済みエンドポイントの作成

API認証済みエンドポイントを設定するには、loco_rsライブラリからcontroller::middlewareをインポートし、middleware::auth::ApiTokenを使用して認証ミドルウェアを関数エンドポイントのパラメータに含めます。

Rustの以下の例を考慮してください。

use loco_rs::prelude::*;
use loco_rs::controller::middleware;
use crate::{models::_entities::users, views::user::CurrentResponse};

async fn current_by_api_key(
    auth: middleware::auth::ApiToken<users::Model>,
    State(_ctx): State<AppContext>,
) -> Result<Response> {
    format::json(CurrentResponse::new(&auth.user))
}

pub fn routes() -> Routes {
    Routes::new()
        .prefix("user")
        .add("/current-api", get(current_by_api_key))
}

API認証済みエンドポイントへのリクエスト

認証済みエンドポイントにリクエストするには、AuthorizationヘッダーにAPI_KEYを渡す必要があります。

Curlリクエストの例:

curl --location '127.0.0.1:5150/api/user/current-api' \
     --header 'Content-Type: application/json' \
     --header 'Authorization: Bearer API_KEY'

API_KEYが有効な場合、ユーザー詳細を含むレスポンスが返されます。
