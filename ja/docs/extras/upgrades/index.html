アップグレード

新しいLocoバージョンがリリースされたらどうすればよいですか？

  • コードリポジトリにクリーンなブランチを作成します。
  • メインの Cargo.toml で Loco のバージョンを更新します
  • CHANGELOGを参照して、行うべき破壊的変更やリファクタリング（もしあれば）を確認してください。
  • プロジェクト内で cargo loco doctor を実行して、アプリケーションと環境が新しいバージョンと互換性があることを確認します。

いつものように、何か問題が発生した場合は、issueを開いて助けを求めてください。

主要なLoco依存関係

Locoは素晴らしいライブラリの上に構築されています。Locoの新しいリリースでのそれらのバージョンと、それぞれの変更履歴に注意を払うことが賢明です。

主なものは以下の通りです:

  • SeaORM, CHANGELOG
  • Axum, CHANGELOG

0.13.x から 0.14.x へのアップグレード

Axum 0.7 から 0.8 へのアップグレード

PR: #1130 Axum 0.8へのアップグレードには破壊的な変更が含まれています。詳細については、アナウンスを参照してください。

アップグレード手順

  • Cargo.toml で、Axum のバージョンを 0.7.5 から 0.8.1 に更新します。
  • use axum::async_trait; を use async_trait::async_trait; に置き換えます。詳細については、こちらを参照してください。
  • URLパラメータの構文が変更されました。更新された構文については、このセクションを参照してください。新しいパスパラメータの形式は次のとおりです: パスパラメータの構文が /:single および /*many から /{single} および /{*many} に変更されました。

boot 関数フックの拡張

PR: #1143

bootフック関数は、追加のConfigパラメータを受け入れるようになりました。関数のシグネチャは以下のように変更されました:

変更前

async fn boot(mode: StartMode, environment: &Environment) -> Result<BootResult> {
     create_app::<Self, Migrator>(mode, environment).await
}

変更後:

async fn boot(mode: StartMode, environment: &Environment, config: Config) -> Result<BootResult> {
     create_app::<Self, Migrator>(mode, environment, config).await
}

必要に応じて Config 型をインポートしてください。

バリデータークレートのアップグレード

PR: #993

Cargo.toml でバリデータークレートのバージョンを更新します:

変更前

validator = { version = "0.18" }

変更後

validator = { version = "0.19" }

truncate および seed フックの拡張

PR: #1158

truncate および seed 関数は、引数として DatabaseConnection の代わりに AppContext を受け取るようになりました。

変更前

async fn truncate(db: &DatabaseConnection) -> Result<()> {}
async fn seed(db: &DatabaseConnection, base: &Path) -> Result<()> {}

変更後

async fn truncate(ctx: &AppContext) -> Result<()> {}
async fn seed(_ctx: &AppContext, base: &Path) -> Result<()> {}

テストへの影響:

seed 関数を含むテストコードも、それに応じて更新する必要があります。

変更前:

async fn load_page() {
    request::<App, _, _>(|request, ctx| async move {
        seed::<App>(&ctx.db).await.unwrap();
        ...
    })
    .await;
}

変更後

async fn load_page() {
    request::<App, _, _>(|request, ctx| async move {
        seed::<App>(&ctx).await.unwrap();
        ...
    })
    .await;
}

0.14.x から 0.15.x へのアップグレード

バリデータークレートのアップグレード

PR: #1199

Cargo.toml でバリデータークレートのバージョンを更新します:

変更前

validator = { version = "0.19" }

変更後

validator = { version = "0.20" }

ユーザークレーム

PR: #1159

  • カスタムユーザークレームの平坦化された (デ)シリアライゼーション: UserClaimsのclaimsフィールドが、Option<Value>からMap<String, Value>に変更されました。

  • generate_token関数における必須のマップ値: generate_tokenを呼び出す際、Map<String, Value>引数が必須となりました。カスタムクレームを使用しない場合は、空のマップ（serde_json::Map::new()）を渡してください。

  • generate_tokenのシグネチャの更新: generate_token関数は、期限切れを参照ではなく値として受け取るようになりました。

ページネーションレスポンス

PR: #1197

ページネーションレスポンスに total_items フィールドが追加され、利用可能なアイテムの総数が提供されるようになりました。

{"results":[],"pagination":{"page":0,"page_size":0,"total_pages":0,"total_items":0}}
