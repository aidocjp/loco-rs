クイックツアー




Locoでブログバックエンドをわずか数分で作成しましょう。まず`loco`と`sea-orm-cli`をインストールします。 
cargo install loco
cargo install sea-orm-cli # Only when DB is needed

これで新しいアプリを作成できます（「`SaaS`アプリ」を選択）。クライアントサイドレンダリング付きのSaaSアプリを選択します: 

✔ ❯ App name? · myapp
✔ ❯ What would you like to build? · Saas App with client side rendering
✔ ❯ Select a DB Provider · Sqlite
✔ ❯ Select your background worker type · Async (in-process tokio async tasks)

🚂 Loco app generated successfully in:
myapp/

- assets: You've selected `clientside` for your asset serving configuration.

Next step, build your frontend:
  $ cd frontend/
  $ npm install && npm run build

次のようなものが得られます:

  • データベースには`sqlite`を使用します。データベースプロバイダについては、モデルセクションのSqlite vs Postgresをご覧ください。
  • バックグラウンドワーカーには`async`を使用します。ワーカー設定については、ワーカーセクションのasync vs queueをご覧ください。
  • クライアントサイドのアセット提供設定。これは、バックエンドがAPIとして機能し、静的なクライアントサイドコンテンツも提供することを意味します。

次に、`myapp`に`cd`し、`cargo loco start`を実行してアプリを起動します。

クライアントサイドのアセット提供オプションを設定している場合は、サーバーを起動する前にフロントエンドをビルドしてください。これは、フロントエンドディレクトリに移動し（`cd frontend`）、`pnpm install`と`pnpm build`を実行することで行えます。

CRUD APIの追加

ユーザー認証機能が生成されたベースのSaaSアプリがあります。`post`と完全なCRUD APIを`scaffold`を使って追加し、ブログバックエンドにしましょう。

それぞれ`-api`、`--html`、`--htmx`フラグを使用して、`api`、`html`、または`htmx`のいずれかのスカフォールドを生成できます。

クライアント向けのクライアントサイドコードベースを持つバックエンドを構築しているため、`--api`を使用してAPIを構築します:

$ cargo loco generate scaffold post title:string content:text --api

選択したスカフォールドテンプレートオプション（`-api`、`--html`、`--htmx`）に応じて、スカフォールドされたリソースを作成する手順は変わります。`--api`フラグまたは`--htmx`フラグを使用する場合は以下の例を使用できます。しかし、`--html`フラグを使用する場合は、ブラウザで投稿作成の手順を実行することをお勧めします。

もし`curl`を使って`--html`スカフォールドをテストしたい場合、デフォルトではContent-Typeを`application/x-www-form-urlencoded`とし、ボディを`title=Your+Title&content=Your+Content`としてリクエストを送信する必要があります。これは、必要であればコード内で`application/json`をContent-Typeとして許可するように変更できます。

次に、`curl`を使って`post`を追加してみましょう。 

$ curl -X POST -H "Content-Type: application/json" -d '{
  "title": "Your Title",
  "content": "Your Content xxx"
}' localhost:5150/api/posts

投稿を一覧表示できます。

$ curl localhost:5150/api/posts

数を数えている人のために――ブログバックエンドを作成するコマンドは次のとおりでした:

  1. `cargo install loco`
  2. `cargo install sea-orm-cli`
  3. `loco new`
  4. `cargo loco generate scaffold post title:string content:text --api`

完了！`loco`で旅を楽しんでください 🚂

SaaS認証を確認する

生成されたアプリには、JWTに基づいた完全に機能する認証スイートが含まれています。

新規ユーザー登録

/api/auth/register`エンドポイントは、アカウント確認のための`email_verification_token`を持つ新しいユーザーをデータベースに作成します。ユーザーには確認リンク付きのウェルカムメールが送信されます。

$ curl --location 'localhost:5150/api/auth/register' \
     --header 'Content-Type: application/json' \
     --data-raw '{
         "name": "Loco user",
         "email": "user@loco.rs",
         "password": "12341234"
     }'

セキュリティ上の理由から、ユーザーが既に登録されている場合、新しいユーザーは作成されず、ユーザーのメール詳細を公開することなく200ステータスが返されます。

ログイン

新しいユーザーを登録した後、以下のリクエストを使用してログインします:

$ curl --location 'localhost:5150/api/auth/login' \
     --header 'Content-Type: application/json' \
     --data-raw '{
         "email": "user@loco.rs",
         "password": "12341234"
     }'

レスポンスには、認証用のJWTトークン、ユーザーID、名前、および確認ステータスが含まれます。

{
    "token": "...",
    "pid": "2b20f998-b11e-4aeb-96d7-beca7671abda",
    "name": "Loco user",
    "claims": null
    "is_verified": false
}

クライアントサイドアプリでは、このJWTトークンを保存し、それを使ってベアラートークン（下記参照）を使用して後続のリクエストを行い、認証されるようにします。

現在のユーザーを取得

このエンドポイントは認証ミドルウェアによって保護されています。以前に取得したトークンを使用して、ベアラートークンの手法でリクエストを実行します（`TOKEN`を以前に取得したJWTトークンに置き換えてください）。

$ curl --location --request GET 'localhost:5150/api/auth/current' \
     --header 'Content-Type: application/json' \
     --header 'Authorization: Bearer TOKEN'

それがあなたの最初の認証済みリクエストになるはずです！

認証ミドルウェアを自分のコントローラーで使用する方法については、`controllers/auth.rs`のソースコードを確認してください。
