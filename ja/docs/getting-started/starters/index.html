スターター

Locoの事前定義されたボイラープレートでプロジェクトのセットアップを簡素化し、開発プロセスをよりスムーズにします。開始するには、CLIをインストールし、ニーズに合ったテンプレートを選択してください。

cargo install loco
cargo install sea-orm-cli # Only when DB is needed

スターターを作成:

✔ ❯ App name? · myapp
✔ ❯ What would you like to build? · Saas App with client side rendering
✔ ❯ Select a DB Provider · Sqlite
✔ ❯ Select your background worker type · Async (in-process tokio async tasks)

🚂 Loco app generated successfully in:
myapp/

- assets: You've selected `clientside` for your asset serving configuration.

Next step, build your frontend:
  $ cd frontend/
  $ npm install && npm run build

利用可能なスターター

SaaSスターター

SaaSスターターは、UIとREST APIの両方を必要とするプロジェクト向けのオールインクルーシブなセットアップです。UIに関しては、このスターターはクライアントサイドアプリまたは従来のサーバーサイドテンプレート（またはそれらの組み合わせ）をサポートしています。

UI

  • ReactとViteをベースにしたフロントエンドスターター（お好みのフレームワークに簡単に置き換え可能）。
  • フロントエンドビルドを指し、フォールバックインデックスを含む静的ミドルウェア。または、サーバーサイドテンプレート用の静的アセットとして設定することもできます。
  • サーバーサイドテンプレート用に設定されたTeraビューエンジン。i18n設定も含まれています。テンプレートとi18nアセットは assets/ にあります。

REST API

  • サービスの健全性を確認するための ping および health エンドポイント。すべてのエンドポイントは cargo loco routes コマンドで確認できます。
  • ユーザーテーブルと認証ミドルウェア。
  • 認証ロジックとユーザー登録を備えたユーザーモデル。
  • パスワード忘れAPIフロー。
  • ウェルカムメールを送信し、パスワード忘れのリクエストを処理するメーラー。

サーバーサイドテンプレート用アセットの設定

SaaSスターターは、フロントエンドクライアントサイドアセット用に事前に設定されています。写真やスタイルなどのアセットを含むサーバーサイドテンプレートレンダリングを使用したい場合は、アセットミドルウェアを設定できます。

config/development.yaml で、サーバーサイドの設定のコメントを解除し、クライアントサイドの設定をコメントアウトしてください。

    # server-side static assets config
    # for use with the view_engine in initializers/view_engine.rs
    #
    static:
      enable: true
      must_exist: true
      precompressed: false
      folder:
        uri: "/static"
        path: "assets/static"
      fallback: "assets/static/404.html"
    fallback:
      enable: false
    # client side app static config
    # static:
    #   enable: true
    #   must_exist: true
    #   precompressed: false
    #   folder:
    #     uri: "/"
    #     path: "frontend/dist"
    #   fallback: "frontend/dist/index.html"
    # fallback:
    #   enable: false

REST APIスターター

フロントエンドなしでREST APIのみが必要な場合は、REST APIスターターを選択してください。後で気が変わり、フロントエンドを配信することにした場合は、static ミドルウェアを有効にし、設定を frontend ディストリビューションフォルダに指定するだけです。

軽量サービススターター

コントローラーとビュー（応答スキーマ）に焦点を当てた軽量サービススターターは、最小限の機能しか持ちません。データベース、フロントエンド、ワーカー、またはLocoが提供するその他の機能なしでREST APIサービスが必要な場合、これが理想的な選択肢です！
