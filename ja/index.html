始める →

ローカルでアプリを構築し、多くの時間を節約。

SaaSやクラウドサービスは不要。認証、ワーカー、メールなど、
すぐに使える機能で時間、費用、労力を節約。 

impl Model {
  pub async fn find_by_email(db: &DatabaseConnection, email: &str)
  -> ModelResult<Self> {

      Users::find()
        .filter(eq(Column::Email, email))
        .one(db).await?
        .ok_or_else(|| ModelError::EntityNotFound)
  }
  
  pub async fn create_report(&self, ctx: &AppContext) -> Result<()> {
      ReportWorker::perform_later(
        &ctx, 
        ReportArgs{ user_id: self.id }
      ).await?;
  }
}

モデル

SeaORMに支えられたリッチなエンティティでビジネスをモデル化し、SQLの記述を避けます。最高の保守性を実現するために、エンティティ上でリレーション、バリデーション、カスタムロジックを構築します。 

pub async fn get_one(
    respond_to: RespondTo,
    Path(id): Path<i32>,
    State(ctx): State<AppContext>,
) -> Result<Response> {
    let item = Notes::find_by_id(id).one(&ctx.db).await?;
    match respond_to {
      RespondTo::Html => html_view(&item),
      _ => format::json(item),
    }
}

pub fn routes() -> Routes {
    Routes::new()
      .prefix("notes")
      .add("/{id}", get(get_one))
}

コントローラ

Webリクエストのパラメータ、ボディ、バリデーションを処理し、コンテンツを認識したレスポンスをレンダリングします。最高のパフォーマンス、シンプルさ、拡張性のためにAxumを使用しています。 


// Literals
format::text("Loco")

// Tera view engine
format::render().view(v, "home/hello.html", json!({}))

// strongly typed JSON responsed, backed by `serde`
format::json(Health { ok: true })

// Etags, cookies, and more
format::render().etag("loco-etag")?.empty()

ビュー

TeraまたはJSONでサーバーサイドレンダリングされたテンプレートを使用します。Locoはサーバー上でビューをレンダリングすることも、フロントエンドアプリとシームレスに連携することもできます。フルスタックのセットアップを自由に構成できます。 

impl worker::Worker<DownloadArgs> for UsersReportWorker {
    async fn perform(&self, args: DownloadArgs) -> worker::Result<()> {
        let all = Users::find()
          .all(&self.ctx.db)
          .await
          .map_err(Box::from)?;
        for user in &all {
          println!("user: {}", user.id);
        }
        Ok(())
    }
}

デプロイ

ガイド付きCLIインターフェースでデプロイ設定を簡単に生成できます。カスタマイズされたデプロイ設定のために、デプロイオプションから選択します。 

jobs:
  db_vaccum:
    run: "db_vaccum.sh"
    shell: true
    schedule: "0 0 * * *"
    tags: ["maintenance"]

  send_birthday:
    run: "user_birthday_task"
    schedule: "Run every 2 hours"
    tags: ["marketing"]
